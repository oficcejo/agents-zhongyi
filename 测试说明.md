# 实时进度功能测试说明

## 修改内容总结

### 1. 后端修改 (backend/)

#### agents.py
- 在 `TCMAgents` 类的 `__init__` 方法中添加了 `progress_callback` 参数
- 添加了 `_send_progress()` 方法用于发送进度更新
- 在所有7个Agent方法中添加了进度回调：
  - `diagnosis_agent` - 中医辨证分析
  - `ancient_cases_agent` - 古籍病案检索
  - `modern_literature_agent` - 现代文献分析
  - `prescription_master_agent` - 经方大师开方
  - `pharmacist_review_agent` - 药剂师审方
  - `rehabilitation_agent` - 康复理疗方案
  - `report_generator_agent` - 生成诊疗报告

每个Agent在开始处理时发送 "processing" 状态，完成时发送 "completed" 状态。

#### main.py
- 添加了 `StreamingResponse` 导入用于SSE
- 添加了 `asyncio` 和 `defaultdict` 导入
- 创建了 `progress_queues` 字典存储每个会话的进度队列
- 新增 `/api/diagnose/progress/{session_id}` GET端点用于SSE推送
- 修改 `/api/diagnose` POST端点：
  - 生成唯一的 `session_id`
  - 创建进度队列和回调函数
  - 创建带进度回调的TCMAgents实例
  - 返回 `session_id` 给前端

### 2. 前端修改 (frontend/)

#### script.js
- 添加了全局变量 `eventSource` 用于SSE连接
- 修改了 `handleFormSubmit()` 函数：
  - 移除了模拟进度动画
  - 添加了SSE连接逻辑
  - 从API响应中获取 `session_id`
- 新增 `connectProgressStream()` 函数：
  - 使用EventSource连接SSE端点
  - 接收实时进度更新
  - 处理完成和错误状态
- 新增 `updateProgress()` 函数：
  - 根据实际状态更新进度条
  - 显示实时消息
- 新增 `resetAllProgress()` 函数：
  - 重置所有进度条状态
- 保留了 `completeAllProgress()` 函数

## 测试步骤

1. **启动服务器**
   ```bash
   cd backend
   python main.py
   ```
   服务器将在 http://localhost:8000 启动

2. **打开浏览器**
   访问 http://localhost:8000

3. **填写测试数据**
   - 疾病名称：感冒
   - 主诉：发热、咳嗽、咽痛3天
   - 四诊信息：
     - 望诊：面色微红，舌质红，苔薄黄
     - 闻诊：声音嘶哑
     - 问诊：恶寒发热，头痛，口渴
     - 切诊：脉浮数
   - 特殊情况：无

4. **观察进度条**
   点击"开始诊疗"后，观察7个进度条的变化：
   - 每个进度条应该在对应的Agent开始处理时变为"处理中"状态（蓝色，50%进度）
   - 显示实时处理消息（如"正在进行八纲、六经、卫气营血、脏腑辨证..."）
   - 完成后变为"已完成"状态（绿色，100%进度）
   - 进度条的激活顺序应该反映实际的Agent执行顺序

5. **验证实时性**
   - 进度条的更新应该与后端实际处理进度同步
   - 不再是固定的时间间隔模拟
   - 可以在后端控制台看到每个Agent的执行日志

## 预期效果

### 之前（模拟进度）
- 进度条按固定时间间隔更新（0ms, 2s, 2s, 5s, 7s, 7s, 10s）
- 与实际AI处理进度无关
- 用户无法知道当前正在执行哪个Agent

### 现在（实时进度）
- 进度条根据实际AI Agent执行状态更新
- 显示每个Agent的实时处理消息
- 用户可以清楚看到当前正在执行的步骤
- 如果某个Agent处理时间较长，进度条会相应停留更久

## 技术实现

### SSE (Server-Sent Events)
- 使用HTTP长连接从服务器推送实时数据到客户端
- 单向通信（服务器→客户端）
- 自动重连机制
- 文本格式：`data: {json}\n\n`

### 进度数据格式
```json
{
  "step": 1,
  "name": "中医辨证分析",
  "status": "processing",
  "message": "正在进行八纲、六经、卫气营血、脏腏辨证..."
}
```

### 状态说明
- `processing`: Agent正在处理
- `completed`: Agent处理完成
- `done`: 所有Agent完成
- `error`: 发生错误
